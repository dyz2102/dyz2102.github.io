有哪些工作需要明天继续努力?

[Eat2gether] Onboarding文档

发表于 2016-11-28 | 分类于 编程学习 |

Eat2gether project Onboarding

产品是：小组约饭形式的交友平台【约饭社交网站】
产品情境：帮助城市年轻人建立起认识新朋友的场景和渠道，通过良好的饭局体验拉进人与人的距离

Part 1.

Before start date

  1. 建立网站微信公众号、微博、Instagram、Facebook，发布最新官方消息、发推广文
  2. 针对建立邀约、发起申请、完成邀约三大功能在网站发布其运作流程
  3. 准备好调查问卷，访问用户使用时候的不便、不理解、以及对产品的反馈意见
  4. 特邀社交达人嘉宾通过平台发起邀约，选择TA们最感兴趣的餐厅、最活跃的地段、最方便约会的时间
  5. 特邀社交达人和参加者撰写博客发表在公众号以及博客中
  6. 每周三定点发布本周最火邀约
  7. 客服电话预约餐厅

【软文宣传】

Around start date

  1. 提前一天提醒约会小组用户第二天时间地点，点击查看“邀约指南”【如何发起具有吸引力的邀约】
    邀约指南：
    a. 个人照片、生活照【什么样的图片更好】
    b. 详细的个人信息，完善个人信息页
    c. 约饭主题新颖
    d. 聊天内容
    e. 提醒：越详细的个人信息会提高约饭概率
  2. 撰写“邀约指南”确保组织者和参与者都拥有最佳约会体验

Start Date

  1. 提醒约会小组约会时间地点，提醒千万不要迟到
  2. 聊天话题推荐：根据小组兴趣爱好标签、根据过去三天时事趣闻，男士/组织者一定要提前准备有趣的话题
  3. 工作人员与餐厅电话确认预约、座位、人数、菜单、【餐厅/网站小礼品】，提醒
  4. 小组可在聊天室当中对话，选择是否提前提供联系方式
  5. 约会期间工作人员随时待命处理客服问题（找不到人、地点、订单出问题等）
  6. 提醒用户查看客户端信息提醒
  7. 邀约之后进行电话回访（包括被拒绝的用户）【标准化问卷】

The first part

  1. 注册网站，登录
  2. 添加邀约或者申请邀约，具体操作在博客里有说明文档

中间

  1. 在“发起约饭”界面填写邀约细节【表单改为在行板式】
  2. 主题建议格式【在[地点+餐厅名]一场[形容词]的[料理]，例：在国贸北京亮一个完美的牛扒晚餐】
  3. 详细信息格式【后面在想，很重要，比较复杂，参考在行】
  4. 约饭地点【5个餐厅，每周更新，用户可自定义】
  5. 根据约饭地点自动显示地址、电话、图片、地图
  6. 邀约需要系统审核才可发布，不可超过24小时

  7. 邀约通过后可以通过社交平台推广：微信、微博、Facebook、Ins?

  8. 参与者可以看到邀约明细【餐厅图片、发起人图片、简介】，申请邀约

  9. 参与者的个人页面的部分信息可以显示在申请页面下，这个页面展示给申请人preview
    10、以周为单位，每周末的两次邀约，要求发起人每周【四】必须确认名单
  10. 同一时间不能发起一个以上的申请{}
  11. 同一时间如果又申请、又发起，则申请通过后不得通过自己发起的邀约{}
  12. 取消邀约会有信用分惩罚{}

Closing & Followup

  1. 邀约结束后，任意用户需要【尽快】确认邀约，【发起人确认后订单就算完结】
  2. 订单完结后双方可以填写反馈【填写反馈者给予奖励】
  3. 5星好评的发起人应得到相应奖励，如【餐厅优惠券、红酒等】
  4. 首次约饭/参与约饭成功，应有相应奖励，如【餐厅优惠券、红酒等】

Part 2.

在Start date 前，客户会寄信来问你什么问题

  1. 是不是一定得1对1组局？【测试期功能上必须1x1，但可以提前在聊天室里与对方沟通，但强烈建议男女比例1：1，另外对方有权退出】
  2. 发起邀约的审核标准是什么？【我们选的餐厅，如果不是，会按照符合我们所选择餐厅标准来审核地点。时间必须是周末晚餐。发起人必须实名认证】
  3. 餐厅的选择标准是什么？【适合约会的价格人均[200元]，地点在三里屯、CBD商区】
  4. 约会中出现不快怎么办？【请拨打服务热线，我们会根据用户协议对责任方问责】
  5. 晚饭费用应该如何支付？【建议AA，可自行协商】
  6. 平台给予奖励【如红酒】应该如何分配？【我们会尽可能提供人均可分配奖励，如果无法分配请自行协商】
  7. 我发起的邀约不被通过怎么办？【为了提高邀约品质，带来更好用户天，我们会给予修改意见，请根据修改意见修改后重新提交】
  8. 发起的邀约没有人申请怎么办？【请分享到微信、微博社交媒体，我们每周也会推送最有意思的邀约到平台首页、公众号等社交媒体】
  9. 申请的人都不想通过怎么办？【请分享到微信、微博社交媒体，我们每周也会推送最有意思的邀约到平台首页、公众号等社交媒体】
  10. 邀约过期了怎么办？【邀约每周末过期，请于每周4之后发起下一周周末的邀约】
  11. 临时有事想取消怎么办？【如果已经与申请者成功配对，可在24小时以外取消。24小时以内不得取消，取消平台资格】
  12. 推荐的餐厅都不喜欢怎么办？【如果不喜欢可以自行添加提交，我们会尽快审核】
  13. 与其他社交网站区别？【Eat2gether与其他社交网站不同的是，我们的邀约信息绝对真实，所有用户都需经过实名认证，邀约都需经过平台专员审核，确保邀约质量】
  14. 平台安全吗？用户信息贩卖？【用户信息绝对安全，【参考在行】】
  15. 用户信息真实？约饭信息真实？【用户认证、邀约审核】
  16. 付费吗？【全部免费】
  17. 约到的人不靠谱怎么办？【请拨打服务热线，我们会根据用户协议对责任方问责】
  18. 能了解对方详细信息吗？【发起邀约后，其他用户能够申请，在申请提交时能够查看对方详细信息】
  19. 对方真的愿意线下见面吗？【所有邀约均为用户主动发起，平台不会发出虚假邀约或者聘用人来发起邀约】
  20. 能在约饭前和对方聊天吗？【可以，行程邀约订单后，可以发起对话】
  21. 没赴约之前觉得不合适怎么办？【赴约前24小时可取消】
  22. 约饭流程？【发起、申请、通过、见面、完成、反馈】
  23. 地点是否安全？【地点均在热闹市中心，经过平台专人实地考察并审核】
  24. 对方是一个人还是多个人？【平台邀约均为1对1，如有变动对方需提前24小时以上沟通，临时变动有权取消】
  25. 如何提高发起被申请的概率？【请根据《邀约指南》详细填写邀约信息并配上精美图片】
  26. 什么餐厅图片更好？【平台会根据推荐餐厅提供精美餐厅图片，如选择平台推荐以外餐厅，可自行上传图片，推荐图片大小500x600以上】
  27. 什么个人图片更好？【推荐五官清晰的大头照，也可添加丰富精彩的生活照提高通过申请的机会】
  28. 有没有约饭成功案例？【很多，请参考blog】

在Start date当天，客户忘记做什么会让使用者体验搞砸？

  1. 联系不上、迟到、找不到地点、直接不出现、有事早走
  2. 不参与买单【发起人有权利要求所有参与者AA买单】
  3. 发起人/参与者没有任何准备
  4. 发起人/参与者对晚饭话题不敢兴趣

他们最常做了什么正确的事达到很好的体验？

  1. 详尽的邀约信息、精美的图片、好吃的餐厅
  2. 提前准备话题、小礼物
  3. 提前到达餐厅，挑选好的座位
  4. 主动正常付款
  5. 填写真实信息
  6. 仔细筛查约会对象
  7. 认真提供约会后反馈，帮助我们更好的完善邀约体验

他们最常做了什么”错误的事“，结果收到很糟的体验？

  1. 选了难吃、不受欢迎的餐厅
  2. 不好看的图片
  3. 不详细的个人或邀约描述
  4. 填写错误个人信息
  5. 饮食上有忌口没有提前沟通

东西卖出后你如何检验他们”做了正确的事“或”做了错误的事“？

  1. 通过用户回访了解邀约详细体验

前端：

Google Chrome自带的Developer Tools，快捷键是Cmd + Opt + I，或者如果直接想用鼠标找到页面上想要调整的物件可以用Cmd + Opt + C。


当发现页面的东西显示不对的时候，打开Cmd + Opt + C，用鼠标找到想要修改的物件，找到左手边HTML文件里相对应的部分，点击，然后找到右手边”style”部分里面的相对应的CSS代码，可以直接修改定义的色值、大小、排版等，也可以直接在左手边HTML里面修改HTML产生效果变动。
